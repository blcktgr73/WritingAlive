# 제품 명세서: 목표 중심 & 하이브리드 글쓰기 기능 확장

**기능 ID**: OUTCOME-HYBRID-WRITING
**버전**: 1.0
**상태**: 개발 제안
**최종 업데이트**: 2025-11-07
**작성자**: 제품 관리팀

---

## 요약 (Executive Summary)

**목표 중심 & 하이브리드 글쓰기**는 WriteAlive의 현재 씨앗 기반(상향식) 워크플로우에 보완적인 하향식(목표 우선) 및 하이브리드(결합) 접근 방식을 추가합니다. 기존 시스템이 흩어진 아이디어로부터의 탐색적 글쓰기에 탁월한 반면, 이 개선 사항은 살리고 글쓰기의 생성적 철학을 유지하면서 정의된 목표를 향한 의도적 글쓰기를 가능하게 합니다.

**비즈니스 임팩트**: WriteAlive를 "탐색적 에세이 작성"에서 "목표 지향적 문서 작성"으로 확장하여, 살리고 글쓰기를 효과적으로 만드는 낮은 에너지 장벽과 반복적 특성을 유지하면서 전문적 글쓰기 맥락(보고서, 제안서, 구조화된 논증)을 다룹니다.

**전략적 적합성**: "글쓰기 스펙트럼" 완성:
- **상향식** (기존): 씨앗 → 중심 → 글쓰기 (발견 주도)
- **하향식** (신규): 목표 → AI 구조 → 글쓰기 (목표 주도)
- **하이브리드** (신규): 목표 + 씨앗 → 중심 → 글쓰기 (두 가지 장점)

---

## 제품 비전 요약

WriteAlive를 순수 창발적 글쓰기 도구에서 발견과 의도를 모두 지원하는 **양방향 글쓰기 시스템**으로 변환합니다. 사용자가 다음과 같이 질문할 수 있게 합니다:

- **발견 모드**: "흩어진 생각들로 무엇을 쓸 수 있을까?" (기존)
- **의도 모드**: "프로젝트 제안서를 써야 하는데 - 구조를 잡아줘" (신규)
- **하이브리드 모드**: "기존 씨앗을 사용해서 학습 방법론에 대해 쓰고 싶어" (신규)

**핵심 철학 유지**: 목표 주도 모드에서도 글쓰기는 작고 진실한 단계로 진행됩니다. 우리는 경직된 개요로 돌아가는 것이 아니라, 글쓰기를 통해 진화하는 부드러운 구조를 제공합니다.

**혁신**: 살리고 글쓰기 + 목표 인식 = 의도적 발견

## 문제 정의

### 현재 시스템의 한계

**1. 탐색적 글쓰기에는 탁월하지만, 전문적 맥락에는 제한적**

현재 워크플로우 (씨앗 → 중심 → 글쓰기):
- ✅ 완벽한 용도: 개인 에세이, 블로그 포스트, 성찰적 글쓰기, 창의적 탐색
- ❌ 어려움: 업무 보고서, 프로젝트 제안서, 구조화된 논증, 마감 주도 글쓰기
- **격차**: "무엇을 써야 할지 알지만, 거기까지 가는 데 도움이 필요해"에 대한 지원 없음

**예시 문제점**:
> "금요일까지 4분기 제품 전략 문서를 써야 해. 고객 인터뷰 씨앗 10개가 있지만, 무작위로 중심을 발견할 수는 없어 - 시장 분석, 고객 불만 사항, 제안된 솔루션, 성공 지표 같은 특정 섹션이 필요해. WriteAlive를 어떻게 사용하지?"

**2. 발견 주도 방식이 전문가 채택을 제한**

전문 작가들의 피드백:
- "업무 글쓰기에는 너무 개방적이야 - 내 매니저는 특정 결과물을 기대해"
- "개인 에세이에는 살리고 글쓰기를 사랑하지만, 업무에는 쓸 수 없어"
- "창의적 발견과 구조적 요구사항 사이의 균형이 필요해"

**시장 격차**: 잠재 사용자의 70%가 전문 지식 근로자이지만, 현재 시스템은 30%(창의적/학술적 탐색적 글쓰기)만 제공합니다.

**3. 놓친 기회: 사용자들은 이미 글쓰기 목표를 가지고 있음**

사용자 조사 결과:
- 60%의 사용자가 막연한 결과물을 염두에 두고 시작 ("학습에 대해 쓰고 싶어")
- 40%가 구체적인 결과물 보유 ("2페이지 프로젝트 회고록이 필요해")
- 현재 시스템은 두 그룹 모두를 동일한 발견 전용 플로우로 강제
- **놓친 가치**: 살리고 원칙을 유지하면서 목표 지향적 글쓰기를 가속화할 수 있음

### 검증된 사용자 불만 사항

**불만 1: "씨앗도 있고 목표도 있어"**

**사용자 스토리** (학술 연구자):
> "피드백 루프에 관한 논문 15개를 모았어 (씨앗). Chapter 2를 위한 문헌 리뷰를 써야 하는 걸 알아 (결과물). 현재 WriteAlive는 흥미로운 중심을 찾지만, 내 챕터 구조와 맞지 않을 수 있어. 시간 지연 문제에 대한 내 논지를 구체적으로 지원하는 중심을 원해."

**현재 해결책**: 사용자가 수동으로 AI 제안 중심을 필터링하고, 관련 없는 제안을 거부하느라 시간 낭비.

**원하는 경험**: 사용자가 결과물 지정 ("피드백 루프의 시간 지연에 대한 문헌 리뷰"), 시스템이 그 결과물을 제공하는 중심 발견.

---

**불만 2: "무엇을 써야 할지 알지만, 빈 페이지를 마주하고 있어"**

**사용자 스토리** (프로덕트 매니저):
> "잘된 점, 안 된 점, 액션 아이템 섹션이 있는 프로젝트 회고록을 써야 해. 씨앗은 없어 - 그냥 결과물만 있어. 전통적인 도구는 모든 것을 미리 개요 작성하도록 강요하지만(높은 에너지), 살리고 글쓰기는 아직 없는 씨앗이 필요해."

**현재 해결책**: 사용자가 WriteAlive를 포기하고, 전통적인 워드 프로세서를 사용하며, 계획 마비를 경험.

**원하는 경험**: 사용자가 결과물을 정의하고, AI가 결과물에 맞는 시작점을 제안하며, 사용자가 반복적으로 작성 (낮은 에너지 유지).

---

**불만 3: "구조는 필요하지만, 경직된 개요는 아니야"**

**사용자 스토리** (기술 작가):
> "API 문서를 쓰고 있어 (명확한 결과물: 엔드포인트, 인증, 오류 설명). 살리고의 단계별 접근 방식을 사랑하지만, 구조를 그냥 '발견'할 수는 없어 - API 디자인에 의해 미리 결정돼 있어. 필수 구조 내에서 생성적 글쓰기를 어떻게 사용하지?"

**현재 해결책**: 사용자가 수동 개요를 만들고, 살리고 이점을 잃음.

**원하는 경험**: 사용자가 구조적 제약(필수 섹션)을 정의하고, 살리고 원칙(작고 진실한 단계)을 사용하여 각 섹션 작성.

## 솔루션 개요

### 3가지 모드 글쓰기 시스템

**모드 1: 씨앗 기반 글쓰기** (기존 - 변경 없음)
- **입력**: 태그가 있는 흩어진 씨앗 노트
- **프로세스**: AI가 중심 발견 → 사용자 선택 → 문서 생성
- **사용 사례**: 탐색적 에세이, 창의적 글쓰기, 개인 성찰
- **강점**: 최대 창의적 발견, 예상치 못한 연결
- **에너지 장벽**: 최저 (노트에 태그만 달고 발견)

**모드 2: 목표 중심 글쓰기** (신규)
- **입력**: 사용자가 글쓰기 목표/결과물 정의 (텍스트 설명)
- **프로세스**: AI가 구조 제안 → 사용자 확인 → 가이드된 반복 글쓰기
- **사용 사례**: 전문 보고서, 구조화된 문서, 마감 글쓰기
- **강점**: 의도적 방향, 전문적 맥락, 구조적 정렬
- **에너지 장벽**: 낮음-중간 (목표 표현 필요하지만, 전체 개요는 불필요)

**모드 3: 하이브리드 글쓰기** (신규)
- **입력**: 글쓰기 결과물 + 기존 씨앗
- **프로세스**: AI가 결과물을 제공하는 중심 발견 → 사용자 선택 → 문서 생성
- **사용 사례**: 논문 작성, 연구 논문, 목표 + 선행 지식
- **강점**: 두 가지 장점 - 의도적 방향 + 기존 통찰력에 기반
- **에너지 장벽**: 중간 (목표 표현 + 씨앗 수집)

---

### 개념적 모델: 결과물을 "인력(Attractor)"으로

**은유**: 결과물을 관련 중심을 끌어당기는 중력장으로 생각하세요

**씨앗 기반 모드** (인력 없음):
```
씨앗: [A] [B] [C] [D] [E] [F] [G]
           ↘  ↓  ↙
         중심 X (교차 영역)
           ↘  ↓  ↙
         중심 Y (감정적)
```
- 중심이 순수하게 씨앗 관계에서 나옴
- 최대 발견, 최소 방향
- 예상치 못한 탁월함을 찾을 수도, 의도한 주제를 놓칠 수도 있음

**목표 중심 모드** (강한 인력):
```
결과물: "프로젝트 회고록 작성: 성공 + 실패 + 교훈"
    ↓ (AI가 구조 생성)
구조: [성공 스토리] [실패 분석] [액션 아이템]
    ↓ (사용자가 섹션별로 작성)
글쓰기: 단계 1 → 단계 2 → 단계 3 (가이드된 반복)
```
- 구조가 결과물을 제공
- 최소 발견, 최대 방향
- 알려진 목표에 효율적, 덜 창의적

**하이브리드 모드** (균형 잡힌 인력):
```
결과물: "피드백에서 시간 지연에 대한 문헌 리뷰"
씨앗: [논문 A] [논문 B] [논문 C] [경험 D]
    ↓ (AI가 결과물 정렬 중심 발견)
중심:
  - "시간 지연 문제" (결과물 제공, A+B+D 연결)
  - "측정 과제" (결과물 제공, B+C 연결)
    ↓ (사용자가 가장 강한 것 선택)
글쓰기: 의도적 + 근거 있음
```
- 중심이 결과물을 제공하고 씨앗에서 나와야 함
- 균형 잡힌 발견 + 방향
- 학술/전문적 종합에 최적

## 대상 사용자 페르소나 (결과물 중심)

### 1. 목표 지향적 전문가 (주요 - 신규 사용자의 40%)

**프로필**:
- 프로덕트 매니저, 컨설턴트, 전략가, 임원
- 특정 비즈니스 결과를 달성하기 위해 작성
- 촉박한 마감, 이해관계자 기대
- 월 5-15개 업무 문서(보고서, 제안서, 메모, 회고록)

**현재 마찰**:
- 전통적 도구: 개요 마비, 높은 에너지 장벽
- WriteAlive (현재): 너무 개방적, 업무 요구사항과 맞지 않음

**불만 사항**:
- "금요일까지 4개 필수 섹션이 있는 4분기 전략 문서를 써야 해"
- "내 매니저는 특정 결과물을 기대해, 창의적 에세이가 아니야"
- "각 문서 작성 전에 개요 작성에 30분을 낭비해"
- "결과물은 명확한데 실행에 마찰이 있어"

**목표**:
- 전문 문서를 효율적으로 작성 (30-60분)
- 구조적 요구사항과 낮은 에너지 글쓰기의 균형
- 전통적인 개요 작성 없이 품질 유지
- 형식이 아닌 사고에 글쓰기 시간 사용

**성공 시나리오**:
> "월요일 오전 9시: 오후 2시까지 4분기 제품 전략 필요. WriteAlive 열기 → 결과물 정의: '시장 동향, 고객 피드백, 제안된 기능, 성공 지표를 다루는 4분기 전략.' AI가 4섹션 구조 제안. 반복 확장을 사용하여 15분에 섹션 1(시장 동향) 작성. 섹션 2-4가 자연스럽게 따라옴. 오후 12시까지 완료, 전통적 접근 방식 대비 2시간 절약."

**목표 중심 가치**:
- 문서 작성 시간 40-60% 단축 (120분 → 60분)
- 개요 마비 제거
- 조직 기대에 맞는 전문적 출력
- 섹션별 반복을 통해 낮은 에너지 유지

---

### 2. 학술 종합가 (2차 - 신규 사용자의 30%)

**프로필**:
- 대학원생, 구조화된 논문을 쓰는 연구자
- 특정 논증 구조로 문헌을 종합해야 함
- 연구 질문/논문(결과물) + 수집된 논문(씨앗) 보유
- 학기당 2-5개 학술 문서(논문, 챕터, 제안서)

**현재 마찰**:
- 씨앗 전용 모드: 중심이 논문과 맞지 않을 수 있음
- 전통적 모드: 개요 중심, 글쓰기 동력 상실
- 격차: 논문을 제공하는 통찰력을 발견해야 함

**불만 사항**:
- "연구 질문은 알지만, 씨앗 중심이 무작위 방향으로 가"
- "20개의 논문(씨앗)과 Chapter 2 구조 요구사항(결과물)이 있어"
- "연결을 발견하고 싶지만, 논문 프레임워크 내에서"
- "문헌 리뷰는 종합과 논증이 모두 필요해"

**목표**:
- 연구 질문 중심으로 문헌 종합
- 논문 제약 내에서 통찰력 발견
- 논증을 제공하는 문헌 리뷰 작성
- 학술적 엄격함과 창의적 종합의 균형

**성공 시나리오**:
> "Chapter 2 문헌 리뷰 다음 달 마감. 결과물: '시간 지연 문제에 초점을 맞춘 피드백 루프 연구 리뷰.' 씨앗: 논문 18개 + 개인 관찰 5개. 하이브리드 모드가 다른 각도에서 시간 지연을 구체적으로 다루는 3개 중심 발견. 각 중심이 섹션이 됨. 문헌 리뷰가 저절로 작성됨 - 종합되고 논증적."

**하이브리드 모드 가치**:
- 논문 정렬 중심 발견 (무작위 연결 대비)
- 더 빠른 문헌 종합 (3주 → 1주)

## 기능 요구사항

### FR-1: 모드 선택 & 가이드

**FR-1.1: 모드 선택 인터페이스**
- 3개 모드(씨앗 기반, 목표 중심, 하이브리드) 중 선택
- 각 모드에 대한 명확한 설명과 사용 사례
- 3개 질문 퀴즈로 올바른 모드 추천 (90% 정확도 목표)

**FR-1.2: 모드 전환 지원**
- 글쓰기 중 모드 전환 허용
- 예: 씨앗 기반으로 시작 → 목표 정의 → 하이브리드로 전환

**FR-1.3: 모드 관련 도움말**
- 각 모드에 대한 튜토리얼
- 모범 사례 및 예제
- 일반적인 함정 및 해결책

### FR-2: 결과물 정의 & 관리

**FR-2.1: 결과물 입력 인터페이스**
- 자유 텍스트 결과물 설명
- 선택적 필드: 독자, 주제, 길이, 마감일
- 템플릿 라이브러리 (10+ 일반 유형)

**FR-2.2: 결과물 템플릿**
템플릿 카테고리:
- **전문적**: 프로젝트 회고록, 전략 문서, 제안서, 상태 보고서
- **학술적**: 문헌 리뷰, 연구 제안서, 논문 챕터
- **기술적**: API 문서, 튜토리얼, 가이드, 릴리스 노트

각 템플릿 포함:
- 일반적인 섹션 구조
- 섹션별 글쓰기 프롬프트
- 예상 길이/시간
- 예제

**FR-2.3: 결과물 검증**
- 명확성 확인 (결과물이 구체적이고 실행 가능한가?)
- 범위 확인 (너무 넓거나 좁지 않은가?)
- 제안 (결과물 설명을 개선하기 위한 AI 권장 사항)

### FR-3: AI 구조 생성

**FR-3.1: 구조 생성 알고리즘**
입력: 결과물 설명, 독자, 주제, 길이
출력:
- 권장 섹션 (제목, 목적, 예상 길이)
- 섹션별 글쓰기 프롬프트
- 섹션 순서 및 종속성
- 총 예상 시간

**FR-3.2: 구조 편집 가능성**
사용자는 다음을 수행할 수 있어야 함:
- 섹션 추가/제거
- 섹션 제목/목적 편집
- 섹션 순서 재배치
- 프롬프트 사용자 정의
- 전체 구조 재생성

**FR-3.3: 비용 & 성능**
- 구조 생성: <5초
- 비용: $0.005-0.010 (중심 발견의 10배 저렴)
- 응답 캐싱 (동일 결과물에 대해)

### FR-4: 하이브리드 모드 중심 발견

**FR-4.1: 결과물 정렬 중심 발견**
입력: 결과물 + 씨앗 컬렉션
프로세스:
1. 결과물 분석 (핵심 주제, 필수 섹션)
2. 씨앗 분석 (내용, 관계)
3. 결과물을 제공하는 중심 발견
4. 결과물 정렬 점수로 중심 순위 지정

출력:
- 2-5개 중심 (기존 중심 발견과 동일한 형식)
- 각 중심에 대한 결과물 정렬 점수 (0-1)
- 설명: 왜 이 중심이 결과물을 제공하는가
- 연결된 씨앗 및 적용 범위 지표

**FR-4.2: 정렬 점수 알고리즘**
```
결과물_정렬 = (
  0.4 × 주제_관련성 +
  0.3 × 섹션_적용_범위 +
  0.2 × 논증_강도 +
  0.1 × 씨앗_다양성
)
```

- **주제_관련성**: 중심이 결과물 주제를 다루는가?
- **섹션_적용_범위**: 중심이 필수 섹션을 지원하는가?
- **논증_강도**: 중심이 강한 논증을 가능하게 하는가?
- **씨앗_다양성**: 중심이 여러 씨앗을 연결하는가?

**FR-4.3: 적용 범위 지표**
- 결과물 섹션의 X%가 이 중심으로 다루어짐
- Y개 씨앗 중 Z개가 연결됨
- 결과물의 미다루어진 영역 식별

### FR-5: 섹션별 글쓰기 인터페이스

**FR-5.1: 섹션 진행 추적**
- 현재 섹션 표시
- 진행률 표시기 (섹션 2/4, 단어 수)
- 완료된 섹션 체크표시
- 시작되지 않은 섹션 회색 처리

**FR-5.2: 섹션 프롬프트**
각 섹션에 대해:
- 목적 상기 ("이 섹션은...")
- 글쓰기 프롬프트 ("다음에 집중...")
- 예상 길이/시간
- 다음 단계 제안 (살리고 반복)

**FR-5.3: 섹션 간 탐색**
- 이전/다음 섹션 버튼
- 섹션 개요 (모든 섹션, 클릭 가능)
- 완료되지 않은 섹션으로 돌아갈 수 있음
- 섹션 순서 건너뛰기 허용 (유연성)

## 임팩트 요약

### API 임팩트
- 새 명령어: `writealive:select-writing-mode`, `writealive:define-outcome`, `writealive:start-outcome-writing`
- 새 서비스: `OutcomeManager`, `StructureGenerator`, `HybridCenterFinder`
- 확장: `AIService.generateStructure()`, `AIService.findHybridCenters()`

### 사용자 경험 임팩트
- **이전**: 1개 워크플로우 (씨앗 기반) - 30% 시장 (창의적/탐색적 작가)
- **이후**: 3개 워크플로우 - 100% 시장 (창의적 + 전문적 + 학술적)
- **전문 사용자 성장**: +100% (사용자 기반의 20%에서 40%로)
- **완료율**: 70% 목표 중심 (씨앗 기반 60% 대비)

### 비즈니스 임팩트
- 시장 범위를 30%에서 100%로 확장
- 전문 지식 근로자 채택 가능
- 학술 연구자를 위한 논문 정렬 글쓰기
- 살리고 철학을 유지하면서 경쟁력 유지
