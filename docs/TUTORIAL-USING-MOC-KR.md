# WriteAlive 고급 튜토리얼: MOC로 센터 찾기

> 이 튜토리얼은 이미 정리해둔 **MOC (Map of Content)**에서 구조적 센터를 발견하는 고급 워크플로우를 다룹니다.
> "흩어진 씨앗 모으기"보다 **"이미 엮어둔 지식에서 패턴 발견하기"**에 초점을 맞춥니다.

---

## 🎯 이 튜토리얼의 목표

1. **MOC의 새로운 활용법 배우기** - MOC는 단순한 링크 모음이 아니라 지식의 구조를 드러낸다
2. **맥락적 센터 발견** - 이미 연결해둔 노트들에서 숨겨진 구조적 중심 찾기
3. **학술 글쓰기 적용** - 논문, 에세이, 보고서 등 구조화된 글쓰기에 활용
4. **지식 베이스 진화** - 정적인 MOC를 생성적 글쓰기의 출발점으로 전환

---

## 🗺️ MOC란 무엇인가?

**MOC (Map of Content)**는 Obsidian 사용자들이 관련된 노트들을 한 곳에 정리하는 방식입니다.

### 전통적인 MOC 사용법
```markdown
# 2025-11 글쓰기 아이디어 MOC

## 빌 에반스 관련
- [[20251104 빌 에반스 연습 철학]]
- [[20251105 기타 연습 인사이트]]
- [[20251107 나무 성장 은유]]

## 프로그래밍 학습
- [[20251106 코드 리뷰 깨달음]]
- [[20251110 리팩토링 경험]]

## 독서 노트
- [[Nature of Order 발췌]]
- [[Atomic Habits 요약]]
```

이렇게 정리해두면 **"관련 노트들이 모여있다"**는 것은 알 수 있지만:
- ❌ 이 노트들의 **공통된 구조적 패턴**은 무엇인가?
- ❌ 이것들을 엮어서 **어떤 글을 쓸 수 있는가**?
- ❌ 가장 강력한 **연결 중심**은 어디인가?

---

## 📖 시나리오: "한 달간 모은 글쓰기 아이디어를 논문으로"

### 배경 스토리

당신은 대학원생이고, 한 달간 "학습 방법론"에 대한 아이디어를 모아왔습니다:
- 빌 에반스의 연습 철학
- 크리스토퍼 알렉산더의 생성적 패턴
- 자신의 프로그래밍 학습 경험
- 기타 연습에서 얻은 인사이트
- 나무 성장 관찰 일지

이 노트들을 `2025-11-학습방법론-MOC.md`에 정리해뒀습니다. 총 15개의 노트가 3개 섹션으로 분류되어 있습니다.

**문제:**
- 이걸로 논문을 써야 하는데, **어떤 주장(thesis)**을 해야 할지 모르겠다
- 노트들 사이에 **공통 패턴**이 있는 것 같은데 명확하지 않다
- 전통적 방법: 노트 15개를 일일이 다시 읽고 직접 정리 (2-3시간 소요)

**WriteAlive 방법:**
MOC에서 AI가 구조적 센터를 자동으로 발견해줍니다 (5분 소요).

## 🔍 워크플로우: MOC → 센터 발견 → 논문 시작

먼저 WriteAlive가 인식할 수 있도록 MOC를 표시합니다.

### 2단계: Find Centers from MOC 실행

#### 2-1. 명령 실행

**방법 A: Command Palette (추천)**
1. **Ctrl/Cmd + P** 열기
2. `WriteAlive: Find Centers from MOC` 입력
3. MOC 파일 선택 (`2025-11-학습방법론-MOC.md`)

**방법 B: Ribbon Button**
1. 좌측 사이드바의 **🌱 아이콘** 우클릭
2. 메뉴에서 **"🔍 Find Centers from MOC"** 선택
3. MOC 파일 선택

#### 2-2. 처리 과정 (자동)

```
✅ MOC 파일 분석 중... (1-2초)
   → 15개의 연결된 노트 발견

✅ 노트 내용 읽는 중... (2-3초)
   → 총 12,000 단어 추출

✅ AI 구조적 패턴 분석 중... (5-7초)
   → Claude 3.5 Sonnet이 센터 발견

✅ 센터 발견 완료!
   → 3개의 강력한 센터, 2개의 중간 센터 발견
```

## 🆚 비교: "씨앗 모으기" vs "MOC에서 센터 찾기"

### Find Centers from Gathered Seeds (기본 워크플로우)

**장점:**
- ✅ 구조 없이 자유롭게 시작 (낮은 진입장벽)
- ✅ 예상치 못한 연결 발견 (창의성)
- ✅ 산발적 아이디어도 활용 가능

**단점:**
- ⚠️ 씨앗이 너무 다양하면 센터가 약할 수 있음
- ⚠️ 학술적 글쓰기에는 구조가 부족할 수 있음

**사용 시나리오:**
- 블로그 에세이, 개인 성찰, 창의적 글쓰기
- "뭘 쓸지 모르겠는데 일단 시작하고 싶을 때"

---

### Find Centers from MOC (고급 워크플로우)

**장점:**
- ✅ 맥락이 명확함 (MOC 자체가 주제 범위 정의)
- ✅ 더 많은 내용 분석 (15-30개 노트)
- ✅ 학술적/전문적 글쓰기에 적합
- ✅ 이미 정리한 지식을 재활용

**단점:**
- ⚠️ MOC를 미리 만들어둬야 함
- ⚠️ 탐색적 글쓰기보다는 구조화된 글에 유리

**사용 시나리오:**
- 논문, 보고서, 기술 문서
- "관련 자료는 많은데 어떻게 엮을지 막막할 때"
- "한 달간 모은 노트를 글로 정리하고 싶을 때"

---

## 💡 실전 활용 팁

### 1. MOC 설계 전략

#### 주제별 MOC (추천)
```markdown
# 2025-11-학습방법론-MOC
← 명확한 주제 범위, 센터 발견이 쉬움
```

#### 기간별 MOC
```markdown
# 2025-11-Monthly-Notes-MOC
← 다양한 주제 혼재, 예상치 못한 연결 발견 가능
```

#### 프로젝트별 MOC
```markdown
# 논문-학습방법론비교연구-MOC
← 목적 지향적, 논문 작성에 최적
```

---

### 2. 센터 발견 최적화

**MOC 크기:**
- 최소: 5개 노트 (너무 적으면 센터 약함)
- 최적: 10-20개 노트 (가장 강한 센터 발견)
- 최대: 30개 노트 (그 이상은 여러 MOC로 분할 권장)

**노트 품질:**
- 각 노트 최소 100단어 이상 권장
- 단순 북마크 링크보다는 자신의 생각이 담긴 노트가 유리
- 사진/다이어그램도 분석 대상 (이미지 인식)

---

### 3. 반복 워크플로우

#### 패턴 1: MOC → 센터 → 초안 → MOC 업데이트
```
1. MOC에서 센터 발견
2. 글 초안 작성
3. 글 쓰면서 새로운 인사이트 생김
4. 새 인사이트를 별도 노트로 저장
5. MOC에 새 노트 추가
6. 다시 센터 발견 → 더 강한 센터 발견!
```

#### 패턴 2: 여러 센터 병렬 탐색
```
1. 하나의 MOC에서 3개의 강력한 센터 발견
2. 각 센터로 별도 문서 3개 시작
3. 3개 문서를 며칠간 병렬 작성
4. 가장 잘 자라는 문서에 집중
5. 나머지는 나중에 활용
```

## ❓ FAQ

### Q1: "씨앗 모으기"와 "MOC에서 찾기" 중 어떤 걸 써야 하나요?

**A:** 둘 다 사용하세요! 상황에 따라 선택:

| 상황 | 추천 방법 |
|------|-----------|
| 막 생각나는 아이디어로 글 시작 | 씨앗 모으기 |
| 한 달간 모은 노트를 정리하고 싶음 | MOC에서 찾기 |
| 논문 챕터 구조 잡기 | MOC에서 찾기 |
| 블로그 포스트 쓰기 | 씨앗 모으기 |
| 프로젝트 보고서 작성 | MOC에서 찾기 |
| 일기/저널 쓰기 | 씨앗 모으기 |

### Q2: MOC가 너무 크면 어떻게 하나요?

**A:** 3가지 전략:

1. **분할**: 30개 이상이면 2-3개의 작은 MOC로 나누기
2. **섹션별 분석**: MOC의 특정 섹션만 선택하는 기능 (향후 추가 예정)
3. **계층적 MOC**: Meta-MOC → Sub-MOC 구조로 관리

### Q3: 센터가 약하게 나오면?

**원인:**
- MOC의 노트들이 너무 이질적임
- 각 노트의 내용이 너무 짧음 (100단어 미만)
- 단순 링크 모음일 뿐 당신의 생각이 없음

**해결책:**
1. 노트마다 최소 한 문단의 자신의 생각 추가
2. 주제 범위를 좁혀서 더 focused한 MOC 만들기
3. "씨앗 모으기" 방법을 대신 사용

### Q4: Living MOC와 어떻게 다른가요?

**Living MOC:**
- 자동으로 새 씨앗을 MOC에 추가
- MOC를 "유지보수"하는 기능

**Find Centers from MOC:**
- MOC에서 구조적 패턴을 발견
- MOC를 "글쓰기에 활용"하는 기능

→ 함께 사용하면 시너지!

**💡 핵심 메시지:**

MOC는 단순한 링크 모음이 아닙니다.
당신이 이미 엮어둔 지식의 구조를 드러내는 **지도(Map)**입니다.

WriteAlive는 이 지도에서 **숨겨진 중심(Hidden Centers)**을 발견하고,
그것을 **글쓰기의 출발점(Starting Point)**으로 전환합니다.

한 달간 모은 15개의 노트가 하나의 논문 Thesis로 진화하는 순간을 경험하세요.
