# WriteAlive 튜토리얼: 목표 중심 글쓰기 (모드 2)

> 이 튜토리얼은 **목표 중심 글쓰기(Outcome-Driven Writing)**를 시연합니다 — 무엇을 써야 할지는 알지만 빈 페이지 앞에서 막막할 때 사용하는 하향식 접근법입니다.
> 빌 에반스의 철학처럼 **"작은 부분을 완전히 진실하게"** — 하지만 의도적인 구조가 각 단계를 안내합니다.

## 🎯 이 튜토리얼의 목표

1. **글쓰기 목표를 효율적으로 정의하기** - 막연한 의도를 2-3분 안에 명확한 목표로 변환
2. **AI 생성 구조 받기** - 경직된 개요 없이 부드러운 섹션별 가이드 제공
3. **추진력을 유지하며 쓰기** - AI 프롬프트가 각 섹션을 안내하여 30-40분 안에 문서 완성
4. **살리고 원칙 유지하기** - 낮은 에너지 장벽, 작은 진실한 단계, 생성적 성장

---

## 📖 시나리오: Q4 프로젝트 회고록 작성하기

### 배경 스토리
금요일 오후입니다. VP가 월요일까지 Q4 회고록을 요청했습니다.

당신은 정확히 무엇이 필요한지 알고 있습니다:
- 이번 분기 팀의 성과 문서화
- 직면한 도전과제 다루기
- 배운 교훈 공유하기
- Q1 실행 항목 정의하기

**전통적인 방법이라면?**
```
[빈 페이지 앞에서 1시간 응시]
- 구조를 어떻게 짜야 하지?
- 성과부터 시작해야 할까, 도전과제부터?
- 각 섹션을 얼마나 상세하게 써야 하지?
- 잘못 구성하면 어쩌지?

[주말이 망가지고, 여전히 시작 못함]
```

**목표 중심 글쓰기 방법이라면?**
2분 안에 목표 정의. AI가 구조 생성. 섹션별로 작성. 35분 만에 완료.

## 🎯 1단계: 목표 정의하기

### 1-1. 목표 중심 모드 열기

**방법 1: 명령 팔레트**

```
목표: "매니저를 위한 주간 학습 요약"
독자: "직속 매니저"
주제: "새로운 기술, 직면한 도전과제, 다음 주 목표"
길이: 짧게 (1-2 페이지)
```

**2단계: 구조 검토** (1분)
AI가 제안할 수 있습니다:
- 섹션 1: 핵심 학습 (배운 것 3-4가지)
- 섹션 2: 극복한 도전과제
- 섹션 3: 다음 주 목표

**3단계: 작성** (15분)
각 섹션의 프롬프트를 따르세요.

**4단계: 완료!**
18분 만에 전문적인 주간 요약.

## 🤝 모드 결합

WriteAlive는 **3가지 보완적 모드**를 지원합니다:

| 모드 | 최적 용도 | 시작 | 구조 |
|------|----------|------|------|
| **씨앗 기반** | 탐색적 에세이, 창작 글쓰기 | 흩어진 노트 | 발견 → 센터 → 작성 |
| **목표 중심** | 전문 문서, 보고서, 스펙 | 명확한 목표 | 목표 → 구조 → 작성 |
| **하이브리드** | 학술 논문, 연구 종합 | 목표 + 연구 노트 | 목표 + 씨앗 → 정렬된 센터 |

**경험 법칙:**
- 무엇을 쓸지 알지만 연구 없음? → **목표 중심**
- 연구는 있지만 구조 없음? → **씨앗 기반**
- 목표와 연구 둘 다? → **하이브리드**

## 💬 자주 묻는 질문

**Q: AI 구조가 내 요구에 맞지 않으면?**
A: 편집하세요! 섹션에서 [편집 ▼] 클릭, [+ 섹션 추가], 또는 [↕️ 순서 변경]. 항상 당신이 컨트롤합니다.

**Q: 섹션을 건너뛸 수 있나요?**
A: 네! 섹션은 필수가 아닙니다. 필요 없으면 완료로 표시하거나 삭제하세요.

**Q: 예측보다 훨씬 많이 쓰면?**
A: 문제없어요! 예측은 안내일 뿐 한계가 아닙니다. 적절하다고 느껴지는 만큼 쓰세요.

**Q: 내 템플릿을 사용할 수 있나요?**
A: 물론이죠! 문서 완성 후 [템플릿으로 저장]을 클릭해 구조를 재사용하세요.

**Q: 시간 예측이 얼마나 정확한가요?**
A: 평균 10분당 150단어 기준입니다. 속도는 다를 수 있습니다 — 쓰면서 예측이 조정됩니다.

**Q: 학술 글쓰기와 호환되나요?**
A: 네! 구조는 목표 중심으로 사용하고, 다듬는 단계에서 학술 관례(인용, 문단 라벨링)를 적용하세요.
