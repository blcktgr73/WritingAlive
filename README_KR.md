# WriteAlive (살리고 글쓰기 도우미)

> **살리고 글쓰기 방법론 기반 AI 보조 글쓰기 도구**

WriteAlive는 **살리고 글쓰기**를 연습할 수 있도록 돕는 Obsidian 플러그인입니다. **김창준님**이 개발한 생성적이고 반복적인 글쓰기 접근법으로, Christopher Alexander의 "The Nature of Order"에 나오는 전체성(Wholeness), 센터(Centers), 생성적 시퀀스(Generative Sequence) 개념과 Bill Evans의 의도적 연습 철학을 결합했습니다.

## 🌱 핵심 철학

### 전통적 글쓰기의 문제점
- 높은 에너지 장벽: 시작하기 전에 완전한 개요 필요
- 노트에 흩어진 아이디어가 일관된 글이 되지 못함
- 빈 페이지 앞에서 겪는 작가의 블록
- 경직된 구조가 창의적 발견을 죽임

### 살리고 글쓰기의 해법: 세 가지 보완적 모드

WriteAlive는 살리고 원칙을 유지하면서 다양한 맥락에 적응하는 **세 가지 글쓰기 모드**를 지원합니다:

#### 1. **씨앗 기반 글쓰기** (상향식 발견)
- 흩어진 씨앗 노트 시작 → 중심 발견 → 떠오르는 패턴에서 글쓰기
- **최적 용도**: 탐색적 에세이, 창의적 글쓰기, 개인 성찰
- **중심 진화**: 약한 연결에서 반복 글쓰기를 통해 강한 구조적 축으로
- **전체성**: 중심이 흩어진 씨앗을 일관된 서사로 얼마나 잘 통합하는지로 측정

#### 2. **목표 중심 글쓰기** (하향식이지만 생성적 단계로)
- 글쓰기 목표 정의 → AI가 부드러운 구조 제안 → 작은 단계로 섹션별 작성
- **최적 용도**: 전문 보고서, 제안서, 마감 주도 문서
- **중심 진화**: 각 섹션이 전체 목표를 제공하는 자체 마이크로 중심 개발
- **전체성**: 섹션이 내부 일관성을 유지하면서 통합된 목표에 얼마나 기여하는지로 측정

#### 3. **하이브리드 글쓰기** (발견 + 의도)
- 목표 정의 + 씨앗 수집 → AI가 목표 정렬 중심 발견 → 기존 통찰력에 근거한 의도적 방향으로 글쓰기
- **최적 용도**: 학술 논문, 문헌 리뷰, 논문 챕터
- **중심 진화**: 씨앗이 논지를 제공하면서도 예상치 못한 통찰을 드러내는 중심으로 결정화
- **전체성**: 정렬(중심이 목표 제공) × 깊이(중심이 새로운 이해 드러냄)로 측정

**모든 모드에 걸친 핵심 원칙**:
- **낮은 에너지 시작**: 작고 진실한 단계로 시작 (완전한 개요 절대 불필요)
- **생성적 성장**: 구조가 미리 정해지지 않고 글쓰기를 통해 진화
- **중심 기반 진화**: 강한 아이디어가 반복 개선을 통해 자연스럽게 약한 것을 강화
- **전체성 추적**: 구조적 일관성에 대한 지속적인 피드백 (1-10 척도)

### 빌 에반스의 연습 철학에서 영감
> "전체를 막연하게 근사하지 말고, 작은 부분을 완전히 진실하게, 완전히 현실적으로, 완전히 정확하게 해라."

이 철학은 씨앗에서 중심을 발견하든, 목표를 향해 쓰든, 기존 지식을 새로운 목표와 합성하든 모두 적용됩니다.

**맥락에 따라 시작점 선택**:

### 🌱 씨앗 모으기 & MOC 통합 (씨앗 기반 & 하이브리드 모드)

- **씨앗 모으기**: 전체 볼트에서 태그된 아이디어를 자동으로 수집
  - 설정 가능한 씨앗 태그: `#seed`, `#idea`, `#💡`, `#씨앗`, 또는 사용자 정의 태그
  - 여러 태그 지원 (설정된 태그 중 하나라도 일치하는 노트 찾기)
  - 다국어 지원: 한글과 영문 태그가 원활하게 작동
  - **빠른 시작 모드**: 최근 10개 씨앗을 자동 선택하여 즉시 센터 발견

- **MOC 통합**: Map of Contents 노트에서 글쓰기 시작
  - `#moc` 태그, 폴더 패턴, YAML frontmatter와 함께 작동
  - **신규: MOC에서 센터 찾기** - 정리된 노트 컬렉션에서 구조적 센터 발견
    - MOC의 10-30개 연결된 노트에 대한 AI 분석
    - MOC 구조 인식 (제목, 교차 도메인 패턴)
    - 최적 MOC 품질을 위한 유효성 검사 경고
    - 비용 투명 분석 (MOC당 $0.020-0.035)
    - MOC 출처 속성이 포함된 문서
  - 튜토리얼 참조: [English](docs/TUTORIAL-USING-MOC-EN.md) | [한국어](docs/TUTORIAL-USING-MOC-KR.md)

- **살아있는 MOC**: 특정 주제와 일치하는 새 씨앗으로 MOC 자동 업데이트
  - MOC별 태그 설정: 예) `seed_tags: [creativity, practice, 창의성]`
  - 세 가지 모드: 실시간, 일일 알림, 수동
  - 수동으로 작성한 내용은 절대 덮어쓰지 않음

- 마찰 없는 경로: "읽으면서 메모" → "일관된 에세이"
- 기존 Zettelkasten, PARA 및 기타 PKM 워크플로우와 호환

### 🎯 AI 보조 중심 발견 (모든 모드)

**중심은 글에 "생명"을 부여하는 구조적 축입니다**. WriteAlive는 반복 글쓰기를 통해 중심이 진화하고 강화되도록 돕습니다.

- **세 가지 발견 워크플로우**:
  1. **흩어진 씨앗에서** (씨앗 기반): 5-15개 태그된 노트에서 패턴 발견
  2. **정리된 MOC에서** (씨앗 기반): 10-30개 연결된 노트를 논제문으로 합성
  3. **목표 + 씨앗에서** (하이브리드): 목표를 제공하면서 통찰을 드러내는 중심 발견

- **중심 진화 추적**:
  - 시각적 강도 표시기: ⭐⭐⭐ 강함, ⭐⭐ 중간, ⭐ 약함
  - 글을 쓰고 다듬으면서 중심이 강화되는 과정 관찰
  - 중심이 씨앗과 연결되고 전체 구조를 지원하는 방식 확인

- **목표 중심 센터 개발** (계획됨):
  - 각 섹션이 전체를 제공하는 마이크로 중심 개발
  - AI가 전체성 지표를 기반으로 심화 또는 확장할 위치 제안

- Claude 3.5 Sonnet, GPT-4, Gemini Pro 지원

### 🌿 생성적 확장 (모든 모드)

- AI가 아이디어를 발전시킬 여러 방향 제안
- 가장 살아있고 전체성을 강화하는 경로 선택
- **씨앗 기반**: 발견된 중심에서 확장
- **목표 중심**: 목표 정렬을 유지하면서 섹션 확장
- **하이브리드**: 발견과 의도의 균형

### 📊 전체성 분석 (모든 모드)

**전체성은 글이 통합된 살아있는 구조로 얼마나 잘 결합되는지 측정합니다**:

- **구조적 일관성 정량화**: 모든 글쓰기 모드에서 1-10 척도
- **중심 진화 추적**: 반복을 통해 중심이 강화되는 과정 확인
- **모드별 지표**:
  - **씨앗 기반**: 중심이 흩어진 씨앗을 얼마나 잘 통합하는지
  - **목표 중심**: 섹션이 통합된 목표에 얼마나 기여하는지
  - **하이브리드**: 정렬(목표 제공) × 깊이(통찰 드러냄)
- 문서 진화에 대한 시각적 피드백
- 전체성 점수를 기반으로 한 반복 개선 가이드

## 🚀 시작하기 (개발자용)

### 사전 요구사항
- Node.js 18+
- Obsidian 1.4.0+

### 설치
```bash
# 저장소 복제
git clone https://github.com/yourusername/WriteAlive.git
cd WriteAlive

# 의존성 설치
npm install

# 플러그인 빌드
npm run build

# 개발 모드 (watch)
npm run dev
```

### Obsidian에서 테스트
1. 플러그인 빌드 (다음 파일들이 생성됨):
   - `main.js` - 플러그인 코드
   - `manifest.json` - 플러그인 메타데이터
   - `styles.css` - 플러그인 스타일
2. 이 3개 파일을 테스트 볼트의 `.obsidian/plugins/writealive/`에 복사
3. Obsidian 설정 → 커뮤니티 플러그인에서 "WriteAlive" 활성화
4. 플러그인 설정에서 AI API 키 구성

**참고**: Obsidian에서 플러그인이 작동하려면 이 3개 파일만 필요합니다.

## 📖 사용 예시

### 방법 1: 씨앗 모으기 (즉흥적 아이디어)

**주말 - 모바일 + 데스크탑 워크플로우**:

📱 **토요일 아침** (지하철, 책 읽기):
- 책에서 영감을 주는 인용구 발견
- Obsidian Mobile 열기 → "Quick Seed" 탭
- 페이지 사진 + 음성: "알렉산더 센터 개념"
- 자동 태그: `#seed #reading` → 5초 만에 저장

📱 **토요일 오후** (카페, 영상 시청):
- 빌 에반스 인용구 듣기
- 빠른 메모: "근사보다 진실" `#idea #practice`
- 오프라인 모드 → 동기화 대기열

📱 **일요일 아침** (공원 산책):
- 나무가 자라는 모습 관찰 → 음성 메모
- "나무는 줄기에서 바깥으로 자란다" `#씨앗 #자연관찰`

💻 **일요일 저녁** (데스크탑, 글쓰기 준비):
1. 모바일 씨앗 3개 자동 동기화됨
2. 명령: `WriteAlive: Gather Seeds`
3. 모든 씨앗 발견 (한글 `#씨앗` + 사진도!)
4. 씨앗 선택 → AI가 센터 제안 → 30분 → 600단어 초안

**설정**: 구성된 태그: `seed, idea, 씨앗, 💡`
**크로스 디바이스**: 모바일에서 캡처한 씨앗이 데스크탑에서 즉시 사용 가능
